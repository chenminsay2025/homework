## 一、部署架构

```
浏览器 https://homework.meituyin.cn
         │
         ▼
┌─────────────────────────────────────┐
│  Nginx（宝塔）                        │
│  ├─ /           → frontend/dist     │  静态 HTML / JS / CSS
│  └─ /api/*      → 127.0.0.1:8002    │  FastAPI 后端
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  PM2 进程 homework-api               │
│  uvicorn app.main:app --port 8002   │
│  读取项目根目录 .env                  │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  SQLite: backend/homework.db        │
│  附件:   backend/uploads/           │
└─────────────────────────────────────┘
```

前端 API 基址为 `/api`（`frontend/src/api.ts`），后端路由也以 `/api` 开头（如 `/api/health`）。**Nginx 必须把 `/api` 完整转发到后端，不能剥掉前缀。**

## 四、`.env` 完整配置

### 4.1 文件位置

```
/www/wwwroot/homework.meituyin.cn/.env
```

与 `backend/`、`frontend/` **同级**，不是放在 `backend/` 里（放根目录即可，`backend/app/config.py` 会自动加载）。

### 4.2 创建方式

```bash
cd /www/wwwroot/homework.meituyin.cn
cp .env.example .env
nano .env
```

或在宝塔 **文件** 中直接编辑。

### 4.3 生产环境完整示例（SQLite）

```env
# ========== 数据库 ==========
# SQLite（单机、小流量推荐，无需额外安装数据库）
DATABASE_URL=sqlite:///./backend/homework.db

# PostgreSQL（多人、长期生产可选，启用时注释掉上面 SQLite 行）
# DATABASE_URL=postgresql://homework:你的数据库密码@localhost:5432/homework

# ========== 运行模式 ==========
# false = 必须登录（生产环境）
# true  = 免登录开发（仅本地，生产切勿开启）
DEV_MODE=false

# ========== JWT 令牌 ==========
# 生产环境务必改为随机长字符串（至少 32 字符）
# 可用命令生成：openssl rand -hex 32
JWT_SECRET=7da5f089013980cfd896c4997fa249f6c3f88ead9908813f733fa5c1767f87c2

# 可选：登录 token 有效期（分钟），默认 10080 = 7 天
# JWT_EXPIRE_MINUTES=10080

# ========== 初始管理员 ==========
# 仅在首次启动、数据库中尚无 admin 用户时自动创建
ADMIN_USERNAME=admin
ADMIN_PASSWORD=请改为强密码

# ========== 跨域 ==========
# 生产环境务必填写你的 HTTPS 域名（不要带末尾斜杠）
# 多个域名用英文逗号分隔
CORS_ORIGINS=https://homework.meituyin.cn
```

### 4.4 各变量说明

| 变量 | 必填 | 默认值（未设置时） | 说明 |
|------|------|-------------------|------|
| `DATABASE_URL` | **是** | `postgresql://homework:homework@localhost:5432/homework` | 漏写会尝试连 PostgreSQL，未安装则启动失败 |
| `DEV_MODE` | 建议 | `true` | 生产必须 `false` |
| `JWT_SECRET` | **是** | 弱默认值 | 用于签发登录 token，`install.sh` 可自动生成 |
| `JWT_EXPIRE_MINUTES` | 否 | `10080`（7 天） | 登录有效期 |
| `ADMIN_USERNAME` | 否 | `admin` | 首次创建的管理员账号 |
| `ADMIN_PASSWORD` | **是** | `admin123` | 首次创建的管理员密码 |
| `CORS_ORIGINS` | 建议 | `*`（允许所有） | 生产应写具体域名 |

### 4.5 PostgreSQL 版 `.env` 示例（可选）

若已安装 PostgreSQL 并创建库：

```env
DATABASE_URL=postgresql://homework:强密码@localhost:5432/homework
DEV_MODE=false
JWT_SECRET=随机长字符串
ADMIN_USERNAME=admin
ADMIN_PASSWORD=强密码
CORS_ORIGINS=https://homework.meituyin.cn
```

**PostgreSQL 建库参考：**

```bash
sudo -u postgres psql
CREATE USER homework WITH PASSWORD '强密码';
CREATE DATABASE homework OWNER homework;
\q
```

### 4.6 `.env` 常见问题

| 问题 | 说明 | 处理 |
|------|------|------|
| 漏写 `DATABASE_URL` | 回退到 PostgreSQL 默认连接 | 补上 SQLite 或 PG 连接串 |
| 重复两行 `DATABASE_URL` | 易改错 | 只保留一行 |
| `CORS_ORIGINS` 被 `#` 注释 | 默认 `*`，不安全 | 取消注释并填域名 |
| 改 `ADMIN_PASSWORD` 不生效 | 仅首次建 admin 时写入 | 登录后改密，或删 `homework.db` 重建 |
| 改 `.env` 后行为未变 | 未重启后端 | `pm2 restart homework-api` |

> **安全：** `.env` 含密钥，勿提交 Git，勿在公开场合粘贴真实密码。

## 六、PM2 完整配置

### 6.1 为什么不使用宝塔「添加 Node 项目」

| 项目 | 类型 | 说明 |
|------|------|------|
| cert、id-svg | Node.js | 可用宝塔 PM2 界面添加 |
| **homework-api** | **Python uvicorn** | 必须用命令行 + ecosystem 文件 |

### 6.2 启动命令

```bash
cd /www/wwwroot/homework.meituyin.cn
pm2 start deploy/pm2.ecosystem.config.cjs
pm2 save
```

### 6.3 PM2 配置文件完整内容（`deploy/pm2.ecosystem.config.cjs`）

```javascript
/** PM2 配置：在项目根目录执行 pm2 start deploy/pm2.ecosystem.config.cjs */
const path = require("path");

const root = path.resolve(__dirname, "..");
const backend = path.join(root, "backend");

module.exports = {
  apps: [
    {
      name: "homework-api",                              // PM2 进程名
      cwd: backend,                                    // 工作目录
      script: path.join(backend, ".venv", "bin", "uvicorn"),  // Python 虚拟环境里的 uvicorn
      args: "app.main:app --host 127.0.0.1 --port 8002",      // 仅本机监听，由 Nginx 反代
      interpreter: "none",                             // 直接执行 uvicorn 二进制，不用 node
      autorestart: true,
      max_restarts: 10,
      min_uptime: "10s",
      env: {
        NODE_ENV: "production",
      },
    },
  ],
};
```

### 6.4 字段说明

| 字段 | 值 | 说明 |
|------|-----|------|
| `name` | `homework-api` | `update.sh` 重启时用此名称 |
| `cwd` | `backend/` | uvicorn 从此目录启动 |
| `script` | `.venv/bin/uvicorn` | 必须先运行 `install.sh` 创建 venv |
| `args` | `app.main:app --host 127.0.0.1 --port 8002` | 绑定本机 8002，不对外暴露 |
| `interpreter` | `none` | 关键：否则会当 Node 脚本执行 |

### 6.5 常用 PM2 命令

```bash
pm2 list                              # 查看所有进程
pm2 describe homework-api             # 查看详情
pm2 restart homework-api              # 重启（改 .env 后执行）
pm2 stop homework-api                 # 停止
pm2 logs homework-api --lines 50      # 查看日志
pm2 save                              # 保存进程列表
pm2 startup                           # 开机自启（按提示执行 sudo 命令）
```

## 七、Nginx / 宝塔完整配置

Nginx 需在 **三个位置** 分别配置，**不要混在一个编辑器里**。

| 配置位置 | 宝塔路径 | 写什么 |
|----------|----------|--------|
| ① 网站根目录 | 网站 → 设置 → **网站目录** | `frontend/dist` 路径 |
| ② 反向代理 | 网站 → 设置 → **反向代理** → 配置文件 | **仅** `location /api { ... }` |
| ③ 主站点配置 | 网站 → 设置 → **配置文件** | `root`、`try_files`、`client_max_body_size` |

---

### 7.1 ① 网站根目录

**宝塔路径：** 网站 → `homework.meituyin.cn` → 设置 → **网站目录**

| 字段 | 填写值 |
|------|--------|
| 网站目录 / 根目录 | `/www/wwwroot/homework.meituyin.cn/frontend/dist` |
| 运行目录 | 同上（或与根目录一致） |

> 必须是 **`frontend/dist`**（npm build 产物），**不是** `/www/wwwroot/homework.meituyin.cn` 项目根目录。

---

### 7.2 ② 反向代理（仅 `/api`）

**宝塔路径：** 网站 → 设置 → **反向代理** → 添加反向代理

#### 界面表单填写

| 字段 | 填写值 | 说明 |
|------|--------|------|
| 代理名称 | `homework` | 随意 |
| 代理目录 | `/api` | **必须是 /api，不是 /** |
| 目标 URL | `http://127.0.0.1:8002` | 后端地址 |
| 发送域名 | `$host` 或 `homework.meituyin.cn` | 不要用 `127.0.0.1` |
| 开启代理 | 开 | |
| 开启缓存 | **关** | API 不要缓存 |
| 高级功能 | 关 | |

#### 反向代理配置文件完整内容（正确 ✅）

点击反向代理右侧 **配置文件**，内容应为：

```nginx
#PROXY-START/api

location ^~ /api
{
    proxy_pass http://127.0.0.1:8002;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
    proxy_connect_timeout 60s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
}

#PROXY-END/
```

**文件路径（参考）：**

```
/www/server/panel/vhost/nginx/proxy/homework.meituyin.cn/*.conf
```

#### ⚠️ 错误示例（会导致 Not Found）

```nginx
# ❌ 错误 1：proxy_pass 末尾有 /，会剥掉 /api 前缀
location ^~ /api {
    proxy_pass http://127.0.0.1:8002/;   # /api/health → /health → 404
}

# ❌ 错误 2：代理整站
location ^~ / {
    proxy_pass http://127.0.0.1:8002;    # 首页也走 API，只看到 JSON
}

# ❌ 错误 3：在反向代理文件里写 root（与主配置冲突）
root /www/wwwroot/homework.meituyin.cn/frontend/dist;  # 报 duplicate root
```

#### `proxy_pass` 路径对照表

| 配置 | 浏览器请求 | 实际转发到后端 | 结果 |
|------|-----------|---------------|------|
| `location /api` + `proxy_pass http://127.0.0.1:8002;` | `/api/health` | `/api/health` | ✅ |
| `location /api` + `proxy_pass http://127.0.0.1:8002/;` | `/api/health` | `/health` | ❌ Not Found |
| `location /api/` + `proxy_pass http://127.0.0.1:8002/api/;` | `/api/health` | `/api/health` | ✅ |

若宝塔保存后自动加尾斜杠，改用第三种写法。

#### 此文件里不要写

- `root`
- `index`
- `location / { try_files ... }`

这些属于主站点配置（下一节）。

---

### 7.3 ③ 主站点配置文件

**宝塔路径：** 网站 → `homework.meituyin.cn` → 设置 → **配置文件**

这是整站 `server { ... }` 块。宝塔会自动生成 SSL、日志等，你只需确认/修改以下关键项。

#### 需要修改的关键行

在 `server { ... }` 内找到并修改为：

```nginx
index index.html;
root /www/wwwroot/homework.meituyin.cn/frontend/dist;
```

在 `access_log` 之前添加（若尚无 `location /`）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}

client_max_body_size 100m;
```

建议注释掉 PHP（本项目不需要）：

```nginx
# include enable-php-00.conf;
```

#### 主配置完整参考（含宝塔自动生成部分）

以下为整合后的参考结构（SSL 证书路径以宝塔实际为准）：

```nginx
server
{
    listen 80;
    listen 443 ssl;
    listen 443 quic;
    listen [::]:443 ssl;
    listen [::]:443 quic;
    http2 on;
    listen [::]:80;
    server_name homework.meituyin.cn;

    # ===== 关键：前端静态目录 =====
    index index.html;
    root /www/wwwroot/homework.meituyin.cn/frontend/dist;

    # ===== SSL（宝塔自动生成，勿删） =====
    #CERT-APPLY-CHECK--START
    include /www/server/panel/vhost/nginx/well-known/homework.meituyin.cn.conf;
    #CERT-APPLY-CHECK--END

    include /www/server/panel/vhost/nginx/extension/homework.meituyin.cn/*.conf;

    ssl_certificate    /www/server/panel/vhost/cert/homework.meituyin.cn/fullchain.pem;
    ssl_certificate_key    /www/server/panel/vhost/cert/homework.meituyin.cn/privkey.pem;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3;
    ssl_ciphers EECDH+CHACHA20:EECDH+CHACHA20-draft:EECDH+AES128:RSA+AES128:EECDH+AES256:RSA+AES256:EECDH+3DES:RSA+3DES:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    add_header Strict-Transport-Security "max-age=31536000";
    error_page 497  https://$host$request_uri;

    error_page 404 /404.html;

    # ===== 引用反向代理（/api → 8002） =====
    include /www/server/panel/vhost/nginx/proxy/homework.meituyin.cn/*.conf;

    # 本项目不需要 PHP，建议注释
    # include enable-php-00.conf;

    include /www/server/panel/vhost/rewrite/homework.meituyin.cn.conf;

    location ~ ^/(\.user.ini|\.htaccess|\.git|\.env|\.svn|\.project|LICENSE|README.md)
    {
        return 404;
    }

    location ~ \.well-known{
        allow all;
    }

    # ===== 关键：SPA 路由 + 上传大小 =====
    location / {
        try_files $uri $uri/ /index.html;
    }

    client_max_body_size 100m;

    access_log  /www/wwwlogs/homework.meituyin.cn.log;
    error_log   /www/wwwlogs/homework.meituyin.cn.error.log;
}
```

#### 主配置各段说明

| 配置项 | 作用 |
|--------|------|
| `root .../frontend/dist` | 首页、JS、CSS 从此目录提供 |
| `index index.html` | 默认首页 |
| `include .../proxy/.../*.conf` | 引入反向代理（`/api` 段） |
| `location / { try_files ... }` | React 路由刷新不 404 |
| `client_max_body_size 100m` | 允许上传最大 100MB 附件 |
| `include enable-php-00.conf` | 可注释，本项目无 PHP |

---

### 7.4 独立 Nginx 配置参考（非宝塔环境）

若不用宝塔，可直接使用项目内 `deploy/nginx.example.conf`：

```nginx
# deploy/nginx.example.conf
server {
    listen 80;
    server_name YOUR_DOMAIN;

    root /www/wwwroot/homework/frontend/dist;
    index index.html;

    client_max_body_size 100m;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }
}
```

HTTPS 在宝塔「SSL」申请证书后通常自动追加 `listen 443 ssl` 配置。

## 九、本地数据迁移（打包上传覆盖）

将本机已有用户、计划、课表、任务及附件迁移到服务器，推荐 **本地打包 → 宝塔上传覆盖** 方式。

### 9.1 需要迁移的文件

| 内容 | 本地路径 | 说明 |
|------|----------|------|
| 数据库 | `backend/homework.db` | 用户、计划、任务等全部业务数据 |
| 附件 | `backend/uploads/` | 整个文件夹（PDF、Word、图片等） |
| WAL 辅助文件（若有） | `backend/homework.db-wal`、`backend/homework.db-shm` | 与本机 `homework.db` 同目录，一并打包 |

> **不要上传本地 `.env`**：服务器保留自己的 `.env`（域名、CORS、JWT 等）。

### 9.2 本机：停服务并打包

**1. 先停本地后端**（避免数据库被占用、复制不完整）：

```bat
scripts\kill-backend.bat 8002
```

或关闭 `start.bat` 打开的后端窗口。

**2. 打包数据**（任选一种）：

**方式 A：只打包数据（推荐，体积小）**

在本机项目目录下，将以下内容打成 zip，例如 `homework-data.zip`：

```
homework-data.zip
├── homework.db
├── homework.db-wal      （若存在）
├── homework.db-shm      （若存在）
└── uploads/
    └── （各计划 ID 子目录及附件）
```

**方式 B：连同代码一起打包**

双击 `scripts\deploy-pack.bat` 生成部署 zip，再 **手动确认** zip 内包含 `backend/homework.db` 与 `backend/uploads/`（脚本会尝试复制数据库，但 `uploads` 需自行确认是否在包内）。

### 9.3 服务器：上传并覆盖

**宝塔 → 文件** → 进入：

```
/www/wwwroot/homework.meituyin.cn/backend/
```

1. 上传 zip → **解压**到 `backend/` 目录。
2. 覆盖以下文件/目录：
   - `homework.db` → `backend/homework.db`
   - `uploads/` → `backend/uploads/`（整个文件夹覆盖）

若只上传了 loose 文件，确保最终结构为：

```
/www/wwwroot/homework.meituyin.cn/backend/homework.db
/www/wwwroot/homework.meituyin.cn/backend/uploads/...
```

### 9.4 重启 PM2（必须）

覆盖数据库后 **必须重启后端**，否则仍可能读写旧连接或旧缓存，导致 401「账号不存在或已禁用」。

SSH 或宝塔 **终端** 执行：

```bash
cd /www/wwwroot/homework.meituyin.cn

# 若进程已在运行（常见）
pm2 restart homework-api

# 若之前停过，用 start 也可以
# pm2 start homework-api
```

验证：

```bash
curl -s http://127.0.0.1:8002/api/health

# 可选：确认用户已在库中
sqlite3 backend/homework.db "SELECT id, username, is_active FROM users;"
```

### 9.5 浏览器：清登录缓存后重新登录

数据替换后，浏览器可能仍保存着 **旧 token**（对应迁移前的服务器用户），需清除后重新登录：

1. 打开 `https://homework.meituyin.cn`
2. F12 → **Application** → **Local Storage** → 删除：
   - `homework_token`
   - `homework_current_plan_id`
3. **Ctrl+F5** 强刷
4. 用 **本地使用过的账号密码** 登录（密码以数据库为准，不是 `.env` 里的 `ADMIN_PASSWORD`，除非本地就是用该密码注册的 admin）

### 9.6 数据迁移步骤速查

```
本机：
  [ ] scripts\kill-backend.bat 8002
  [ ] 打包 homework.db + uploads（+ wal/shm 若有）
  [ ] 上传到宝塔 backend/ 并覆盖

服务器：
  [ ] 覆盖 backend/homework.db
  [ ] 覆盖 backend/uploads/
  [ ] pm2 restart homework-api          ← 必须
  [ ] curl health 正常
  [ ] sqlite3 确认 users 表有本地用户

浏览器：
  [ ] 删除 homework_token、homework_current_plan_id
  [ ] 重新登录，检查计划与附件
```

### 9.7 迁移常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| 401「账号不存在或已禁用」 | 覆盖 db 后未重启 PM2，或浏览器旧 token | `pm2 restart homework-api` + 清 localStorage 重登 |
| 登录密码错误 | 库未覆盖成功，仍是服务器旧库 | 重新上传 `homework.db` 并 restart |
| 计划有数据，附件打不开 | 未上传 `uploads/` | 补传 `backend/uploads/` 并 restart |
| 数据看起来不对 | 上传到错误目录 | 确认路径为 `.../backend/homework.db` |

### 10.2 Git Push 后宝塔自动拉取（Webhook）

实现：**本机 `git push` → GitHub 通知宝塔 → 宝塔拉代码 → 执行部署脚本 → 重建前端 + 重启 PM2**。

> **不会覆盖的数据**（已在 `.gitignore`）：`.env`、`backend/homework.db`、`backend/uploads/`、`frontend/dist/`（由 `update.sh` 重新构建）。

#### 第一步：宝塔配置 Git 拉取

**路径：** 网站 → `homework.meituyin.cn` → 设置 → **Git 拉取**

| 字段 | 填写值 |
|------|--------|
| 仓库地址 | `git@github.com:chenminsay2025/homework.git` |
| 分支 | `main` |
| 拉取目录 | `/www/wwwroot/homework.meituyin.cn`（站点根目录） |

点击 **生成/查看 SSH 公钥**，复制整段公钥（以 `ssh-rsa` 或 `ssh-ed25519` 开头）。

#### 第二步：GitHub 添加 Deploy Key

1. 打开 https://github.com/chenminsay2025/homework/settings/keys
2. **Add deploy key**
3. Title：`baota-homework`
4. Key：粘贴宝塔 SSH 公钥
5. **Allow write access**：可不勾（只拉代码不需要写权限）
6. 保存

回到宝塔，点击 **拉取** 测试，应能成功拉下代码。

#### 第三步：宝塔填写 Webhook 与部署脚本

仍在 **Git 拉取** 页面：

1. 复制页面上的 **WebHook 地址**（形如 `http://服务器IP:端口/hook?access_key=...` 或带域名的 URL）
2. **部署脚本** 填写（路径按你的站点目录）：

```bash
cd /www/wwwroot/homework.meituyin.cn && bash scripts/deploy-webhook.sh
```

`deploy-webhook.sh` 会：`git pull` → 执行 `update.sh`（装依赖、构建前端、`pm2 restart homework-api`）。

3. 保存

> 若宝塔版本在 Webhook 触发时 **已自动 git pull**，部署脚本也可简化为：
> ```bash
> cd /www/wwwroot/homework.meituyin.cn && bash scripts/update.sh
> ```
> 推荐仍用 `deploy-webhook.sh`，避免部分版本未自动拉取。

#### 第四步：GitHub 添加 Webhook

1. 打开 https://github.com/chenminsay2025/homework/settings/hooks
2. **Add webhook**
3. **Payload URL**：粘贴宝塔 WebHook 地址
4. **Content type**：`application/json`
5. **Secret**：留空（除非宝塔要求填写）
6. **Which events**：选 **Just the push event**
7. **Active**：勾选
8. 保存

推送后：

2. 宝塔 → Git 拉取 → 查看 **拉取日志** / 部署日志

#### Webhook 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| GitHub Delivery 红色失败 | Webhook URL 错、服务器防火墙未放行宝塔端口 | 检查 URL；安全组/防火墙放行面板端口 |
| Delivery 成功但代码未更新 | 部署脚本路径错或未执行 | 检查脚本绝对路径；SSH 手动跑一遍 |
| `git pull` 失败 Permission denied | Deploy key 未配或未授权 | 重新添加 SSH 公钥到 GitHub |
| `npm run build` 失败 | Node 版本低或内存不足 | 宝塔装 Node 18+；必要时加 swap |
| 代码更新了但页面没变 | 浏览器缓存 | Ctrl+F5；或看 `frontend/dist` 修改时间 |
| push 后数据库/附件丢了 | 误提交了 db/uploads | 它们应在 `.gitignore`，勿 `git add` 这些文件 |
| 部署失败 `dubious ownership` | 目录属主与 git 执行用户不一致 | 见下方 **10.3 节** |

#### 10.3 部署失败：`dubious ownership`

日志类似：

```text
fatal: detected dubious ownership in repository at '/www/wwwroot/homework.meituyin.cn'
```

**原因：** 站点目录最初是 zip 上传解压的（属主多为 `root`），宝塔 Git 拉取时用其他用户执行 git，Git 2.35+ 会拦截。

**在服务器 SSH 执行（推荐两条都做）：**

```bash
# 1. 允许该目录（root 用户）
git config --global --add safe.directory /www/wwwroot/homework.meituyin.cn

# 2. 若宝塔用 www 用户跑 git，也加一条
sudo -u www git config --global --add safe.directory /www/wwwroot/homework.meituyin.cn

# 3. 可选：把整个站点目录属主改为 www（与 Nginx/PHP 常见用户一致）
chown -R www:www /www/wwwroot/homework.meituyin.cn
```

然后回到宝塔 **Git 管理 → 仓库**，再点 **拉取** 或 **部署记录** 里重试。

验证：

```bash
cd /www/wwwroot/homework.meituyin.cn
git pull origin main
```

无报错即修复成功。

## 十一、常见问题排查

| 现象 | 原因 | 处理 |
|------|------|------|
| `127.0.0.1:8002/api/health` 正常，域名返回 `{"detail":"Not Found"}` | Nginx `proxy_pass` 末尾有 `/`，剥掉了 `/api` | 见 [7.2 节](#72--反向代理仅-api) |
| 浏览器打开域名只有 JSON | 整站 `location /` 被反代到 8002 | 删除整站反代，只保留 `/api` |
| 首页空白或 403/404 | `root` 未指向 `frontend/dist` | 改网站目录 |
| 登录页能开，刷新子路由 404 | 缺少 `try_files` | 主配置加 `location / { try_files ... }` |
| 保存 Nginx 报 `root directive is duplicate` | 在反向代理文件里写了 `root` | 反向代理文件只写 `location /api` |
| PM2 启动后立刻退出 | 缺 `.env`、`DATABASE_URL`、或未 `install.sh` | 检查配置并重新安装 |
| 401「账号不存在或已禁用」 | 迁移数据后未 `pm2 restart`，或浏览器旧 token | 见 [9.4、9.5 节](#94-重启-pm2必须) |
| 登录密码不对 | admin 已用旧密码创建，或库未覆盖 | 后台改密，或重新上传 `homework.db` 并 restart |
| 上传失败 413 | 上传大小限制 | 确认 `client_max_body_size 100m` |
| 改 `.env` 不生效 | 未重启后端 | `pm2 restart homework-api` |
| `pm2 save` 后重启服务器进程消失 | 未配置开机自启 | `pm2 startup` 并按提示执行 |
