# 作业计划表 · 安装说明

本文说明如何在 **本机**、**其它 Windows 电脑** 以及 **服务器** 上安装与运行。

---

## 一、环境要求

| 组件 | 版本建议 |
|------|----------|
| 操作系统 | Windows 10/11（本机/其它电脑）；Linux 可用于服务器 |
| Python | 3.10 及以上 |
| Node.js | 18 及以上（含 npm） |
| 数据库 | **本地试用**：SQLite（无需安装）<br>**生产/多人**：PostgreSQL 14+ |
| 可选 | Docker Desktop（仅在使用 PostgreSQL 容器时需要） |

---

## 二、本机安装（Windows）

### 1. 获取代码

- 解压备份包 `homework-backup-*.zip` 到任意目录，例如 `H:\homework`。
- 或使用 git 克隆项目到该目录。

### 2. 配置环境变量

```bat
copy .env.example .env
```

用记事本编辑 `.env`，本地单机推荐：

```env
DATABASE_URL=sqlite:///./backend/homework.db
DEV_MODE=false
JWT_SECRET=请改为至少32位随机字符串
ADMIN_USERNAME=admin
ADMIN_PASSWORD=请改为强密码
```

> `DEV_MODE=true` 时可免登录开发，**正式使用请设为 false**。

### 3. 一键启动

双击项目根目录 **`start.bat`**：

- 自动安装 Python / npm 依赖（首次较慢）
- 后端：`http://localhost:8002`
- 前端：`http://localhost:5173`
- 会尝试释放 5173、8002 端口并显示局域网访问地址

浏览器打开：**http://localhost:5173/login**

### 4. 手动启动（可选）

**后端：**

```bat
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8002
```

**前端（新开终端）：**

```bat
cd frontend
npm install
npm run dev
```

## 三、安装到其它 Windows 电脑

1. **复制整个项目文件夹**（或解压备份 zip）到目标电脑，路径尽量不要含特殊字符。
2. 目标电脑安装 **Python 3.10+**、**Node.js 18+**（安装时勾选 “Add to PATH”）。
3. 复制 `.env.example` 为 `.env` 并按第二节配置。
4. 若需 **迁移数据**：
   - SQLite：复制 `backend/homework.db` 和 `backend/uploads/` 文件夹。
   - PostgreSQL：在新库执行备份恢复（见第五节）。
5. 双击 **`start.bat`** 启动。
6. 局域网访问：其它设备浏览器输入 `http://该电脑IP:5173`；若无法访问，在 Windows 防火墙中放行 **5173**、**8002** 端口（`start.bat` 会尝试调用 `scripts\open-lan-ports.bat`）。

## 四、服务器部署（Linux 生产环境）

### 快速流程（宝塔推荐）

> **详细步骤、各配置文件完整内容与踩坑说明** 请参阅 **[deploy/宝塔服务器部署.md](./deploy/宝塔服务器部署.md)**（含 `.env`、PM2、Nginx 主配置、反向代理、Webhook 等）。

**在你电脑上：**

1. 先停本地后端：`scripts\kill-backend.bat 8002`（迁移数据前必须）。
2. 双击 **`scripts\deploy-pack.bat`**，生成 `deploy\homework-deploy-日期时间.zip`；或单独打包 `backend\homework.db` 与 `backend\uploads\` 为数据 zip。
3. 若数据库目录下有 `homework.db-wal`、`homework.db-shm`，一并打入 zip。

**在服务器上（摘要）：**

1. 代码放到站点目录（Git 拉取或解压 zip），例如 `/www/wwwroot/homework.meituyin.cn`。
2. 复制 `.env.example` 为 `.env` 并编辑（`DEV_MODE=false`、强密码、`CORS_ORIGINS` 填你的域名，**勿漏 `DATABASE_URL`**）。
3. `bash scripts/install.sh`
4. `pm2 start deploy/pm2.ecosystem.config.cjs && pm2 save`（**勿**在宝塔 PM2 里按 Node 项目添加）
5. Nginx：**根目录** → `frontend/dist`；**反向代理** 仅 `/api` → `127.0.0.1:8002`（`proxy_pass` **无尾斜杠**）；主配置加 `try_files` 与 `client_max_body_size 100m`
6. **迁移本地数据（打包上传覆盖）：** 宝塔文件 → `backend/` → 上传并覆盖 `homework.db`、`uploads/`，然后 **`pm2 restart homework-api`**（必须）；浏览器清除 `homework_token` 后重新登录。
7. 自检：`curl -s https://你的域名/api/health` 应返回 `"status":"ok"`，浏览器能打开登录页。

**后续更新代码：** 本机 `git push origin main`，宝塔 Webhook 自动拉取并部署（见 **[deploy/宝塔服务器部署.md](./deploy/宝塔服务器部署.md)** 第 10.2 节）；或 SSH 手动 `bash scripts/update.sh`。

> 本地数据迁移（打包上传覆盖）见 **[deploy/宝塔服务器部署.md](./deploy/宝塔服务器部署.md)** 第九节。

### 1. 系统准备

```bash
# 示例：Ubuntu
sudo apt update
sudo apt install -y python3 python3-pip python3-venv nginx
# Node 18+ 见 https://nodejs.org 或 nvm
```

### 2. 数据库（PostgreSQL）

**方式 A：Docker**

```bash
cd /opt/homework
docker compose up -d
```

**方式 B：云数据库**  
在阿里云 RDS、腾讯云、Supabase、Neon 等创建 PostgreSQL，记下连接串。

`.env` 示例：

```env
DATABASE_URL=postgresql://用户:密码@数据库主机:5432/库名
DEV_MODE=false
JWT_SECRET=生产环境随机长密钥
ADMIN_USERNAME=admin
ADMIN_PASSWORD=强密码
```

### 3. 后端

```bash
cd /opt/homework/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 测试运行
uvicorn app.main:app --host 0.0.0.0 --port 8002
```

生产环境建议用 **systemd** 或 **supervisor** 守护进程，例如 systemd 单元：

```ini
[Unit]
Description=Homework Planner API
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/homework/backend
EnvironmentFile=/opt/homework/.env
ExecStart=/opt/homework/backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8002
Restart=always

[Install]
WantedBy=multi-user.target
```

首次启动会自动 **建表、迁移、创建管理员**。

### 4. 前端（构建静态文件）

```bash
cd /opt/homework/frontend
npm install
npm run build
```

构建产物在 `frontend/dist/`。生产环境需让 API 与前端同域或配置反向代理：

**Nginx 示例：**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /opt/homework/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        client_max_body_size 100m;
    }
}
```

> 上传附件需调大 `client_max_body_size`；上限还可在运营后台 **设置** 中配置。

HTTPS 建议使用 Let’s Encrypt（certbot）。

### 5. 附件目录

确保运行后端的用户对 `backend/uploads/` 有写权限；备份时需一并备份该目录。

## 五、数据备份与恢复

| 内容 | 路径/方式 |
|------|-----------|
| SQLite 数据库 | `backend/homework.db`（备份前请 **关闭后端**，或运行 `scripts\backup-to-chenmin.bat`） |
| PostgreSQL | `pg_dump` / 云控制台备份 |
| 上传附件 | `backend/uploads/` |
| 配置 | `.env`（含密钥，勿泄露） |

**SQLite 恢复：** 停止服务 → 替换 `homework.db` → 重启。  
**迁移到 PostgreSQL：** 导出 SQL/CSV 后导入，或在新环境重新建计划（小数据量时可选手工）。

## 六、端口与防火墙

| 服务 | 默认端口 | 说明 |
|------|----------|------|
| 前端开发 | 5173 | `npm run dev`，局域网访问用 |
| 后端 API | 8002 | 前端通过 Vite 代理 `/api` |
| PostgreSQL | 5432 | Docker 或远程库 |

生产环境对外通常只暴露 **80/443**（Nginx），8002 仅本机监听。

## 七、更新版本

1. 备份 `.env`、`backend/homework.db`（或数据库）、`backend/uploads/`。
2. 用新版本覆盖代码（保留上述文件）。
3. 重新安装依赖并重启：

```bat
cd backend && pip install -r requirements.txt
cd ..\frontend && npm install && npm run build
```

4. 启动后迁移脚本会自动执行（`app/migrate.py`）。

## 八、故障排查

| 现象 | 处理 |
|------|------|
| 前端能开但 API 报错 | 确认后端 8002 已启动；`vite.config.ts` 代理指向 8002 |
| **`/api/plans/.../day-items` 404（Not Found）** | 多为 **8002 端口仍被旧版后端占用**，新后端没绑上端口。运行 `scripts\kill-backend.bat 8002` 后重新 `start.bat`。浏览器访问 `http://localhost:8002/api/health` 应返回 `"status":"ok"` 且 `features` 含 `day-items` |
| **`/api/admin/deleted` 404** | 同上，旧后端未包含删除记录功能。确认 `http://localhost:8002/api/health` 的 `features` 含 **`deleted-records`**（`api_version` 应为 `0.3`） |
| **`start.bat` 提示 Backend not ready** | 旧 uvicorn 进程（尤其带 `--reload` 的）在 Windows 上可能残留。先运行 `scripts\kill-backend.bat 8002`，关闭所有 Backend 窗口，再重新 `start.bat` |
| 404 且 detail 为「计划不存在」 | 未复制 `backend/homework.db`，或浏览器 localStorage 仍记着旧计划 ID；复制数据库后刷新，或在首页重新选计划 |
| 405 / 404 上传附件 | 重启后端加载最新路由 |
| 端口被占用 | 运行 `scripts\kill-port.bat 8002` 或 5173 |
| pip / npm 很慢 | 配置国内镜像源 |

**迁移后自检（推荐）：**

1. 浏览器打开 `http://localhost:8002/api/health` — 应看到 JSON，且 `features` 含 `day-items`、`deleted-records`（`api_version`: `0.3`）。
2. 若只有 `{"detail":"Not Found"}`，说明 8002 上不是本项目后端，请关掉占用端口的进程后重新 `start.bat`。
3. 需要旧数据时，务必一并复制 `backend/homework.db` 与 `backend/uploads/`（见第三节第 4 步）。
